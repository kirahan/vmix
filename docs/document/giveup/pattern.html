vmix is a webapp view-model component FrontEnd framework


view-model is interactive mode between dom-data ,the job is fulfill and package

1,vm just like normal html element,include attribute,style,event.
2,write html-style-script in one file.
3,fulfill interaction between dom and data.
4,vmix will automatic finish all the lifecycle,like load,parse,dependency,dom,event bind and response.

component is organization mode of webapp ,the job is module reuse and interaction

1,extend reuse: include prototype vm,modify it and automatic generate new one.
2,inject reuse: include one vm,inject something to it,and control the event.
3,api interaction:  vm.children[index].abc(),vm.getgvm(id).abc(),just like document.getElementById()
4,message interaction:vm.pub(channel,message,cb)

webapp between user-(dom-data)-server,the job is global control

global message event,
global data flow,
global route,
global log,debug,error catch,
